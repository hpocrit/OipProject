## Задание 2 — Токенизация и лемматизация

### Описание

Модуль извлекает текст из скачанных HTML-страниц, разбивает на слова (токены),
очищает от мусора и группирует по леммам (базовым формам слов).

### Зависимости

- `beautifulsoup4` — извлечение текста из HTML
- `pymorphy3` — лемматизация русского языка

Все зависимости указаны в `requirements.txt`.

### Установка

```bash
pip install -r requirements.txt
```

### Запуск

```bash
python task2/tokenizer.py
```
### Выходные файлы

**tokens.txt** — один токен на строку, без дубликатов:
```
алгоритм
программирование
компьютер
```

**lemmas_tokens.txt** — лемма и все её словоформы через пробел:
```
алгоритм алгоритм алгоритма алгоритмов алгоритму
программирование программирование программирования
```

### Примеры использования

```bash
# Запуск обработки (HTML-файлы берутся из ./pages/)
python task2/tokenizer.py

# Проверить количество токенов
wc -l task2/tokens.txt

# Проверить формат лемм
head task2/lemmas_tokens.txt
```

## Задание 3 — Инвертированный индекс и булев поиск

### Описание

Модуль строит инвертированный индекс по леммам из задания 2 и выполняет
булев поиск по нему. Поддерживаются операторы AND, OR, NOT и произвольная
вложенность скобок.

### Зависимости

- `pymorphy2` — лемматизация поисковых запросов

### Запуск

**Шаг 1** — построить индекс (нужно выполнить один раз):

```bash
python task3/build_index.py
```

**Шаг 2** — поиск:

```bash
# Интерактивный режим
python task3/search.py

# Одиночный запрос из аргумента
python task3/search.py "авокадо AND NOT манго"
```

### Структура файлов после запуска

```
task3/
├── build_index.py          # построение инвертированного индекса
├── search.py               # булев поиск
├── inverted_index.json     # индекс (машиночитаемый)
└── inverted_index.txt      # индекс (человекочитаемый)
```

### Формат inverted_index.txt

```
авокадо: 32, 39, 54, 69
август: 8, 9, 10, 12, 15, 27, ...
рецепт: 1, 2, 3, 4, 5, ...
```

Каждая строка — лемма и список номеров документов, в которых она встречается.

### Примеры запросов

```
авокадо AND манго
рецепт OR суп
NOT борщ
(авокадо AND манго) OR (кефир AND йогурт)
```

Термины запроса автоматически нормализуются к лемме, поэтому форма слова
не важна (поиск по «рецептов» найдёт то же, что и «рецепт»).

## Задание 4 — TF-IDF

### Описание

Модуль вычисляет метрики TF (term frequency) и IDF (inverse document frequency)
для каждого токена и каждой леммы по всей коллекции документов из задания 1.
Словари терминов и лемм берутся из результатов задания 2.

**Формулы:**
- `TF(t, d) = count(t, d) / total_tokens(d)`
- `IDF(t) = log( N / df(t) )`, где N — число документов, df — количество документов, содержащих термин

### Зависимости

- `beautifulsoup4` — извлечение текста из HTML

### Запуск

```bash
python task4/tf_idf.py
```

### Структура файлов после запуска

```
task4/
├── tf_idf.py
├── tf_idf.log                     # лог выполнения
├── tokens_idf.txt                 # глобальный IDF для всех токенов
├── lemmas_idf.txt                 # глобальный IDF для всех лемм
├── tokens/
│   ├── 001_tokens_tf_idf.txt
│   ├── 002_tokens_tf_idf.txt
│   └── ...
└── lemmas/
    ├── 001_lemmas_tf_idf.txt
    ├── 002_lemmas_tf_idf.txt
    └── ...
```

### Формат выходных файлов

**tokens_idf.txt** / **lemmas_idf.txt** — глобальный IDF по всей коллекции:
```
токен	IDF
авокадо	3.218876
рецепт	0.116534
```

**\<doc\>_tokens_tf_idf.txt** / **\<doc\>_lemmas_tf_idf.txt** — TF, IDF и TF·IDF для каждого термина документа:
```
токен	TF	IDF	TF-IDF
авокадо	0.002134	3.218876	0.006869
рецепт	0.000276	0.116534	0.000032
```

Строки с нулевым количеством вхождений в документ не записываются.
